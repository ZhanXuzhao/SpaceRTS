# Space RTS — Dev Road Map

> 基于 Godot 4 的太空即时战略游戏（2D 俯视视角）

---

## 🎯 项目概况

一款类《家园》(Homeworld) 风格的太空 RTS 原型，支持框选单位、攻击指令、多种武器系统、小地图导航等核心 RTS 操作。当前聚焦于**战斗系统**和**操控手感**的打磨。

---

## ✅ 已完成功能

### 核心 RTS 操控
| 功能 | 描述 |
|------|------|
| 框选 | 左键拖拽矩形框选己方单位 |
| 点击选 | 左键单击选中单个单位 |
| 右键移动 | 右键空地 → 移动 |
| A+左键攻击移动 | 移动到目标点，沿途自动攻击敌人 |
| A+左键区域攻击 | 对指定半径范围（500px）内敌人自动攻击 |
| 右键攻击 | 右键点击敌方 → 锁定攻击 |
| 反打机制 | 被攻击时自动反击攻击者 |
| ESC 暂停菜单 | 暂停/继续/重新开始/退出 |

### 战斗系统
| 功能 | 描述 |
|------|------|
| 4 种武器 | 子弹（BULLET）、导弹（MISSILE）、激光（LASER）、近防炮（PD） |
| 武器槽位 | 最多 8 个槽位，独立冷却、独立旋转 |
| 槽位偏移 | 每个槽位有位置偏移（上/下/左/右/对角线） |
| 槽位独立瞄准 | 每个炮塔从槽位实际位置计算瞄准角度 |
| 子弹物理 | 带质量/加速度/最高速度的物理运动 |
| 追踪导弹 | 自动追踪目标，持续转向 |
| PD 拦截 | 近防炮自动拦截敌方追踪弹 |
| 激光 | 连续光束（3 层辉光效果） |
| 护盾/结构 | 双血条系统（护盾自动恢复，结构受击永久损失） |

### 单位系统
| 功能 | 描述 |
|------|------|
| 队伍系统 | BLUE（玩家）/ RED（AI） |
| 碰撞分离 | 单位之间自动排斥避免重叠 |
| 飞船朝向 | Sprite2D 旋转跟随移动速度方向 |
| 选中高亮 | 选中单位有视觉效果 |
| 血条/护盾条 | 头顶显示，护盾蓝条在上，结构彩色条在下 |

### 相机与地图
| 功能 | 描述 |
|------|------|
| 滚轮缩放 | Camera2D 平滑缩放 |
| 边缘滚动 | 鼠标移到屏幕边缘自动平移 |
| 小地图 | 右下角动态小地图，点击/拖拽跳转 |
| 无限地图 | 无边界限制，小地图动态适配 |
| 全屏模式 | 启动自动全屏 |

### 环绕模式（W）
| 功能 | 描述 |
|------|------|
| W+左键 | 选中单位环绕目标飞行 |
| 环绕方向 | 根据切入位置自动选择顺时针/逆时针 |
| 角速度 | 自动根据半径计算，保证飞船能追上轨道 |
| 取消环绕 | 右键移动或新指令取消 |
| 轨道绘制 | 调试时可看到轨道圆 |

### UI 系统
| 功能 | 描述 |
|------|------|
| 暂停覆盖层 | CanvasLayer Button 控件 |
| 游戏结束 | 胜负判定 + 重启/退出 |
| 攻击光标 | A 模式下红色十字准星 |
| 环绕光标 | W 模式下绿色圆圈 |

## 🏗️ 技术架构

```
project.godot
├── assets/
│   ├── ship2.svg          ← 飞船纹理（朝右）
│   ├── ship.svg
│   ├── missile.svg        ← 导弹纹理（朝右）
│   └── missile.png
├── scenes/
│   ├── main.tscn          ← 主场景（入口）
│   ├── unit.tscn          ← 单位预制体（Area2D + Sprite2D + CollisionShape2D）
│   └── projectile.tscn    ← 弹体预制体（Area2D + Sprite2D）
└── scripts/
	├── game_config.gd      ← 全局数值配置（常量）
	├── main.gd             ← 主循环（输入、相机、单位管理、绘制覆盖）
	├── unit.gd             ← 单位（属性、战斗、移动、炮塔、绘制）
	├── projectile.gd       ← 弹体（物理运动、追踪、伤害）
	├── weapon.gd           ← 武器数据（Resource 类型）
	├── minimap.gd          ← 小地图
	└── overlay.gd          ← 暂停/菜单 UI
```

### 关键设计决策
- **Unit 自包含**：AI 逻辑目前在 `main.gd` 中，后续可抽离独立 AI 控制器
- **数值集中**：`game_config.gd` 统一管理所有游戏数值
- **Weapon 为 Resource**：武器是纯数据，无节点开销
- **Sprite2D 渲染**：飞船和弹体使用 Sprite2D 而非 `_draw`，旋转不影响子节点

## 🚧 未来规划

### Phase 1 — 视觉与体验优化（短期）

- [ ] **爆炸/受击特效**：粒子系统或 Sprite 动画实现爆炸、护盾闪避
- [ ] **激光光束视觉**：激光目前 draw 在 Unit 上，改为独立 beam 节点
- [ ] **音效系统**：开火、爆炸、移动提示音
- [ ] **选中圈优化**：更明显的选中圆形指示器
- [ ] **士气/状态图标**：单位状态可视化的角标
- [ ] **拖拽滚动速度**：适应当前缩放级别的平移速度

### Phase 2 — 游戏机制扩展（中期）

- [ ] **AI 系统**：独立 AI 控制器，支持多种战术行为（进攻/防守/巡逻）
- [ ] **阵型系统**：单位群组移动时保持阵型
- [ ] **装备自定义**：开战前配置武器槽位组合
- [ ] **护盾/装甲类型**：不同武器对不同护盾/装甲的伤害修正
- [ ] **单位升级**：战斗中积累经验，提升属性
- [ ] **残骸 & 回收**：战损单位留下残骸，可回收资源

### Phase 3 — 内容扩展（长期）

- [ ] **多品种单位**：主力舰、护卫舰、战斗机等不同角色
- [ ] **资源系统**：采集资源建造单位
- [ ] **科技树**：升级武器、护盾、引擎等
- [ ] **战役模式**：任务关卡设计
- [ ] **多玩家/网络**：联机对战
- [ ] **地图编辑器**：自定义战场地形

### 待修复 / 已知问题

- [ ] 单位到达目标点后的抖动（`distance_to < 4.0` 条件）
- [ ] PD 光束视觉与开火逻辑的分离可进一步简化
- [ ] 单位数量增多时的性能优化（目标查找遍历 `_all_units`）
